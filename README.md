## Features

- 🛰️ **Sentinel-2 Imagery**: Surface Reflectance, cloud-masked (QA60), median composite
- 🗺️ **Interactive Map**: Leaflet-based viewer with Google Satellite basemap
- 📅 **2026 YTD**: January 1 – Today (updates dynamically when you re-run the script)
- 🎨 **True Color**: RGB composite (B4 Red, B3 Green, B2 Blue)
- ☁️ **Cloud-Free**: Automatic masking of clouds and cirrus
- 📍 **Thailand Boundary**: Built-in FAO/GAUL dataset, visually outlined
- 🔊 **Layer Toggle**: Switch between Sentinel-2 and Google Satellite
- 🎚️ **Opacity Slider**: Adjust S2 layer transparency
- 💚 **100% Free**: No API keys required (Google Satellite XYZ is public)

## Quick Start (3 Steps)

### Clone from GitHub (Optional)

```bash
git clone https://github.com/prametkae-blip/GEE_THA.git
cd GEE_THA
```

Or just download `Sentinel2.html` directly from the repo!

---

### Step 1: Generate Sentinel-2 Tile URL

1. Go to **[Google Earth Engine Code Editor](https://code.earthengine.google.com)**
2. Create a **New Script** (if needed, accept the terms)
3. Open `gee/sentinel2_thailand_2026.js` from this folder
4. Copy the entire script into the Code Editor
5. Click **"Run"**
6. In the **Console** (bottom right), find the line starting with `https://ee-results...`
7. **Copy that entire URL** (it's long)

### Step 2: Paste URL into HTML

1. Open `Sentinel2.html` in a text editor (VS Code, Notepad++, etc.)
2. Find this line (around line 195):
   ```javascript
   const SENTINEL2_TILE_URL = 'https://ee-results.earthengine.app/map/PLACEHOLDER_URL_FROM_GEE/tiles/{z}/{x}/{y}';
   ```
3. Replace `PLACEHOLDER_URL_FROM_GEE` with the URL token you copied from the Console
4. Save the file

### Step 3: Open & Explore

- **Local file**: Double-click `Sentinel2.html` (or right-click → Open with Browser)
- **Web server**: Upload to a web server and open via HTTP/HTTPS
- **No installation** required! Leaflet and tiles load from CDN

## How It Works

### The GEE Script (`gee/sentinel2_thailand_2026.js`)

Runs in **your authenticated** Google Earth Engine account:

1. **Data**: Loads `COPERNICUS/S2_SR_HARMONIZED` (Sentinel-2 Surface Reflectance)
2. **Spatial Filter**: Bounds by Thailand (FAO/GAUL 2015 Level 0)
3. **Temporal Filter**: Date range 2026-01-01 → today
4. **Cloud Mask**: Applies QA60 bitmask (bits 10–11) to remove clouds & cirrus
5. **Composite**: Computes `.median()` to reduce remaining noise
6. **Visualization**: True-color RGB (B4, B3, B2) with dynamic scaling
7. **Export**: Prints the tile URL that Leaflet will request

### The Viewer (`Sentinel2.html`)

- Leaflet map centered on Thailand
- **Basemap**: Google Satellite (public XYZ, no API key)
- **Overlay**: Sentinel-2 tile layer from the GEE URL
- **Boundary**: Thailand outline from GeoJSON (dashed red line)
- **Controls**:
  - Layer toggle (switch S2 on/off)
  - Opacity slider (0–100%)
  - Zoom/pan (standard map controls)

## Important Notes

### Tile URL Expiration

The token in the tile URL (generated by `getMapId()`) is **temporary**:
- ✅ Valid for ~1 hour in **interactive mode** (running script in Code Editor)
- ❌ Expires after that; tiles will stop loading
- 🔄 **Solution**: Re-run the GEE script and update the URL in `Sentinel2.html`

### For Permanent Hosting

If you want a **persistent tile URL** (doesn't expire), use **Earth Engine Apps**:
- Wrap the GEE script in `ui.Map()` and `ui.App()`
- Deploy as a public or private app
- Get a permanent shareable link

This is beyond the scope of the current setup but documented in the GEE script as a comment.

### Date Range

The script uses `ee.Date(Date.now())` to dynamically set the end date to **today**. Re-running the script will automatically extend the composite to the latest available Sentinel-2 data.

## Troubleshooting

### "Tiles won't load" or "Layer is blank"

- ✅ Did you paste the correct tile URL into `Sentinel2.html`?
- ✅ Did you re-run the GEE script recently? (Token might have expired — re-run and update the URL)
- ✅ Check browser Console (F12) for error messages
- ✅ Try zooming/panning the map

### "I don't have a Google Earth Engine account"

- Go to **[code.earthengine.google.com](https://code.earthengine.google.com)**
- Sign in with your Google account
- Accept the terms of service
- You're ready to run GEE scripts!

### "Map doesn't load"

- Is `data/Thailand_Pramet.geojson` present in the folder?
- Are you opening `Sentinel2.html` via HTTP/HTTPS or `file://`? (both should work)
- Check the browser console (F12) for CORS or loading errors

### "Boundary line doesn't show"

- The outline is styled as a **dashed red line** with no fill
- Zoom out or pan around to see it
- It should be visible as a thin red dashed border around Thailand

## Customization

### Change Visualization Bands

In `gee/sentinel2_thailand_2026.js`, modify the `visParams` object:

```javascript
// Currently: true color (B4=Red, B3=Green, B2=Blue)
var visParams = {
  bands: ['B4', 'B3', 'B2'],
  min: 0,
  max: 0.3,
};

// Example: NDVI (Normalized Difference Vegetation Index)
// var visParams = {
//   bands: ['NDVI'],
//   min: -1,
//   max: 1,
//   palette: ['blue', 'white', 'green']
// };
```

Re-run the script and update the URL in `Sentinel2.html`.

### Adjust Opacity in HTML

In `Sentinel2.html`, find this line (around line 195):

```javascript
s2Layer = L.tileLayer(SENTINEL2_TILE_URL, {
  maxZoom: 14,
  minZoom: 2,
  opacity: 0.9,  // <-- change this (0 = transparent, 1 = opaque)
  ...
});
```

Or use the **Opacity Slider** in the sidebar.

### Change Cloud Threshold

In the GEE script, modify the pre-filter:

```javascript
// Currently: exclude images >50% clouds
.filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 50))

// Change to 30% for stricter filtering:
.filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 30))
```
